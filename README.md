## Core Features

### 1. Real-time AI Operations Dashboard
- Live zone heatmap with crowd density percentages
- Emergency banner + AI suggested action per affected zone
- Volunteer activity and system logs timeline
- Node/agent health simulation panel

### 2. Telegram-first Volunteer Workflow
- Guided onboarding: `/start` -> volunteer ID -> zone assignment
- Persistent action menu for updates and emergencies
- Auto-broadcast to all active volunteers
- High-congestion threshold alerts (for example, >85%)

### 3. AI-assisted Incident Intelligence
- Text and image analysis for crowd density + emergency classification
- Structured AI output: `density`, `emergency`, `summary`, `suggestion`
- Automatic Firestore updates for dashboard synchronization
- Fallback keyword/rule engine if AI key is unavailable

### 4. Emergency Orchestration
- Any emergency report is tagged and escalated
- Alert fan-out to all registered volunteers
- Suggested next action generated by AI
- Incident visible immediately on admin view

## Example Emergency Scenario

Input from volunteer (Telegram):
- "Gate A got fire and smoke, crowd is panicking"

StadiumAI flow:
1. AI detects emergency intent and estimates crowd pressure
2. Zone is updated with emergency message + suggestion
3. Dashboard switches to emergency state
4. All active volunteers receive broadcast instantly
5. AI suggests action (example): "Divert crowd to Gate B and dispatch fire response"

## System Architecture

### Frontend
- React + Vite app with two views:
- Admin Command Center: `/`
- User Navigation Demo: `/user`

### Backend
- Express server (`server.ts`) running with Vite middleware in development
- Serves static `dist` in production
- Handles Telegram bot polling + AI processing orchestration

### Data Layer
- Firebase Firestore for real-time shared state
- Collections used:
- `zones`
- `telegram_users`
- `system_logs`
- `broadcast_messages`

### AI Layer
- Primary/fallback model strategy:
- Groq (if `GROQ_API_KEY` present)
- Gemini (if `GEMINI_API_KEY` present)
- Vision analysis supported for volunteer photo reports

## Tech Stack

- Frontend: React 19, Vite, Tailwind CSS, motion/react, lucide-react
- Backend: Node.js, Express, TypeScript (`tsx` runtime)
- Bot: `node-telegram-bot-api`
- Database: Firebase Firestore + Firebase Admin SDK
- AI: `@google/genai` + Groq OpenAI-compatible endpoint
- Deployment target: Google Cloud Run (scripts included)

## Quick Start

### Prerequisites
- Node.js 18+
- npm
- Firebase project with Firestore enabled
- Telegram bot token (for bot features)
- At least one AI API key (`GEMINI_API_KEY` or `GROQ_API_KEY`)

### 1) Install dependencies
```bash
npm install
```

### 2) Configure environment
Create `.env` in project root.

Example:
```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_ADMIN_CHAT_ID=
CRIC_API_KEY=
GEMINI_API_KEY=
VITE_GEMINI_API_KEY=
GROQ_API_KEY=
APP_URL=http://localhost:3000
```

Notes:
- `GEMINI_API_KEY` is used on server-side AI processing.
- `VITE_GEMINI_API_KEY` is used by frontend components needing direct AI calls.
- `TELEGRAM_ADMIN_CHAT_ID` can be used as head-control recipient.

### 3) Configure Firebase
Update `firebase-applet-config.json` with your Firebase web app config and Firestore database ID.

For server-side admin access, either:
- Set `FIREBASE_SERVICE_ACCOUNT` to a service account JSON path, or
- Use Application Default Credentials (recommended for Cloud Run)

### 4) Run locally
```bash
npm run dev
```

App will run on:
- `http://localhost:3000`

### 5) Production run
```bash
npm run build
npm start
```
